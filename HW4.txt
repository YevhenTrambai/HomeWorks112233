Етапи завантаження Linux:
1.Завантаження BIOS (POST):
 BIOS (Basic Input/Output System) — це низькорівневе програмне забезпечення, що зберігається на чіпі пам’яті материнської плати комп’ютера.
При включенні комп’ютера BIOS починає пошук завантажувального пристрою для запуску операційної системи.
Якщо процес ініціалізації BIOS та пошук завантажувального пристрою завершуються успішно, комп’ютер готовий до завантаження операційної системи.
2.Перший етап завантаження (MBR або GPT):
 MBR (Master Boot Record) або GPT (Guided Partition Table) — це завантажувачі першого етапу.
MBR містить таблицю розділів диска і знаходиться в першому секторі завантажувального диска.
Його основне завдання — перехід до розділу диска, з якого необхідно виконати подальший код завантаження операційної системи.
3.Другий етап завантаження (GRUB2):
 Сучасні дистрибутиви Linux використовують Grub (Grand Unified Bootloader) для завантаження.
Grub набагато складніший за LILO (Linux Loader) і дозволяє змінювати конфігурацію завантаження на льоту.
Він знаходить образ ядра Linux на диску та ініціалізує його.
4.Ініціалізація ядра:
 Ядро починає первинну ініціалізацію, налаштовує обробник переривань та ініціалізує необхідні структури даних.
Розпаковує архів initramfs (зазвичай /boot/initramfs-linux.img) у файлову систему в оперативній пам’яті.
5.Ініціалізація головного процесу (init або systemd):
 Ініціалізація головного процесу відбувається після завантаження ядра.
Вона включає запуск системних служб (демонів) та налаштування середовища для користувача.
6.Рівні виконання (runlevels):
 Рівні виконання визначають, які служби запускаються під час завантаження.
Наприклад, рівень 3 — це текстовий режим, а рівень 5 — графічний.

Як переглянути системні журнали?
 Щоб переглянути поточний системний журнал: cat /var/log/syslog
 Для перегляду журналу попередньої сесії: cat /var/log/syslog.1
 За допомогою GNOME Logs.
 Це графічний інструмент для перегляду журналів, який відображає всі журнали, створені для журналів systemd.
 Щоб встановити GNOME Logs у Ubuntu: sudo apt install gnome-logs
 Щоб переглянути журнали ядра: cat /var/log/kern.log
 Для перегляду журналів ядра попередньої загрузки: cat /var/log/kern.log.1
 Журнали сервера відображення X11 можна знайти у /var/log/ або $HOME/.local/share/xorg/.
 Щоб знайти журнали Xorg: locate Xorg.0.log
 Команда dmesg для перегляду повідомлень журнала ядра або “кільцевого буфера” ядра Linux.

-rw------- : Опишіть цей дозвіл. Як додати прапорець виконуваного файлу?
  Дозвіл -rw------- в Linux вказує на файл, до якого є доступ лише для читання та запису власником.
-rw-: Це означає, що власник файлу має права на читання та запис.
-------: Це означає, що ніякі інші користувачі не мають доступу до цього файлу.
Щоб додати прапорець виконуваного файлу, використовуется команда chmod. Щоб додати право виконання для власника, треба виконати:
chmod +x filename
де filename — це назва вашого файлу.

Яка різниця між apt і dpkg?
 Основна відмінність між apt і dpkg полягає у їх рівній ієрархії:
 dpkg — це низькорівневий інструмент, який безпосередньо встановлює, видаляє та керує пакунками на оперативній системі.
Він працює на рівні системи і не має зручного користувацького інтерфейсу.
 apt — це вищорівневий інструмент, який забезпечує користувачам зручний та простий інтерфейс для управління пакунками.
Коли apt ,або apt-get встановлює пакунок, він фактично використовує dpkg на бекенді для цього.
